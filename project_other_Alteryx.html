Alteryx use-case

Basic ETL process using Alteryx

Background

We have been provided with monthly trial balances, but...
... one month = one file.

The Target Group is made of two trading entities: ie we need to keep track of those two trading entities but we also need to aggregate those two to ultimately present a consolidated view.

The task here is to collect the data from multiple files (84 in total, see further details below), treat it, clean it, and summarize that data into a tabular structured format.

Data cleaning is an important first step for almost every data journey. This is a highly underrated skill! This part of the data lifecycle is often the most time consuming (at least something common to all my projects).

We have to ensure the data is: reliable; in a readable format for computers; contains no (minimal) missing values. We can use many tools to do this, a few of which, I have demonstrated below and in this portfolio.

Challenges

🐌 Large number of files

We need to aggregate trial balances for two trading entities from multiple files. The historical period covers FY18 (12), FY19 (12), FY20 (12) and YTD21 (6) ie 42 different files for each entity.

🐌 Unstructured data source: data that needs to be cleaned

✔ Cleaning process is a manual process that would take long to achieve as it would need to be repeated 84 times

✔ Need to add information: calculate the balance (debit – credit) for each period and add an ‘entity column’

🐌 Need to summarize the data into one tabular structure format, rather than 84 different tabs

Typical process using Excel

  1. Open each source file and copy the relevant tab into the main data-book;
  2. Calculate TB’s balance for each period (ie each tab) and make sure the sheet structure is identical for all tabs;
  3. Add an extra column and add the entity name;
  4. Stack all the label and nominal ledger code and then remove duplicates so all the information could be summarised in a single table; and
  5. Create a dynamic excel formulas to pick up all the data into each relevant tab. For instance SUMIFS + INDIRECT that would based on date, label and NL code criteria.

This manual excel work would typically take ½ day / one day, depending on level of experience. Overall this is a lenghthy process which is manual, and hence increases the risk of error.

The same information can be processed into Alteryx within only 30 mins!
